# v0.1 Scope

## Goal

Working aiDeck runtime with project-status renderer, demo mode, help page, and MCP server. Sufficient to dogfood inside atomic-skills development.

## In-scope

### Runtime
- Hono HTTP server on `127.0.0.1:7777` (configurable port)
- chokidar watcher on `.atomic-skills/**`
- SSE endpoint pushing canonical-data change events to browser
- REST API: `GET /api/state/:consumer/:slug?`, `GET /api/help`
- Static asset serving for built Vue client

### Frontend (Vue 3)
- Dark-first theme (CSS variables, light-mode-ready later)
- Project-status renderer:
  - Plan bird's-eye view (phase tree with status icons)
  - Initiative zoom view (stack, tasks, parked, emerged)
  - Phase breadcrumb (`F2/9 · plan: v3-redesign`)
- Help page (auto-discovers skills from `atomic-skills` repo if linked, otherwise static doc)
- Demo mode (seeded fixtures)
- Annotation panel (read-only display; write via MCP in v0.1)
- Highlight indicators on entities

### MCP server
- 18 tools across 5 categories: read, mutate, exit-gate, feedback, meta (see [`mcp-tools.md`](./mcp-tools.md))
- stdio transport (Claude Code, Cursor compatibility)
- Runs alongside HTTP server (same process)
- Tools cover full operational loop: read state → mark done → verify gates → annotate → inbox

### CLI
- `aideck serve [--port=N] [--no-mcp]`
- `aideck demo`
- `aideck mcp` (MCP-only)
- `aideck --help`

### Demo fixtures
- 1 sample plan (3 phases, 12 tasks, mixed states)
- 1 sample initiative (standalone, no parent plan)
- 1 ad-hoc session entry
- 3 sample annotations, 2 highlights

## Out-of-scope for v0.1

- parallel-dispatch / parallel-dispatch-audit renderers → v0.2
- hunt / review renderers → v0.3
- Cross-cutting views (home, timeline, repo health) → v0.4
- Custom consumer registration (third-party tools) → v0.2+
- Light theme polish → v0.2
- Dashboard-side mutation (click to mark task done) → v0.2+
- Authentication / multi-user → never (local-first only)
- Cloud sync → never (local-first only)

## Definition of done

Detailed feature gates live in [`feature-contracts.md`](./feature-contracts.md). v0.1 composite gate:

- [ ] All 13 feature gates (F1-F13) met
- [ ] aiDeck renders real sda-v2 v3-redesign plan (9 phases, ~10 initiatives, 70+ tasks)
- [ ] MCP server exposes 18 tools, callable from Claude Code
- [ ] `npx @henryavila/aideck demo` works from clean machine
- [ ] Unit test coverage ≥ 70% on schemas + server + mcp
- [ ] Manual smoke test against real `.atomic-skills/` repo
- [ ] README + license + npm metadata correct for publish

## Dependencies on atomic-skills

v0.1 of aiDeck requires atomic-skills to evolve in parallel:

1. project-status redesigned to plan/phase/task hierarchy (per design discussion 2026-05-19)
2. Skills frontmatter (YAML) added to all 12 skill files for help-page auto-discovery
3. project-status writes canonical YAML matching `@henryavila/aideck/schemas` shapes

These are tracked separately in atomic-skills initiatives.
